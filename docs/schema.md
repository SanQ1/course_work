Рішення щодо дизайну.
Рішення зробити order як масив order_item's було прийняте для кращої структурованості схеми.
Всі інші таблиці були створені, щоб задовольнити вимоги, поставлені на початку розробки.
Нормалізація усіх таблиць відповідає 3НФ, глибша нормалізація не є необхідною.
Індексація робилась для найбільш ймовірно використовуваних полів, тобто 'email' в customer, та 'name' в product.
